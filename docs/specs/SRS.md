# AI 포털 사이트 프로젝트 요구사항 명세서

## 사용자(users)
- 사용자는 아래 정보를 유지해야 한다.
    - 사용자 번호(id)
    - 이메일(email)
    - 비밀번호(password)
    - 유저네임(username)
    - 이름(name)
    - 역할(role)
    - 상태(status)
    - 현재 플랜 번호(plan_id)
    - 잔여 토큰량(remaining_tokens)
    - 비밀번호 재설정 토큰(reset_token)
    - 토큰 만료 일시(token_expiry)
    - 생성 일시(created_at)
    - 수정 일시(updated_at)
- 사용자는 사용자 번호(id)로 식별한다.
- 사용자는 사이트에 가입하기 위해 이메일(email), 비밀번호(password), 유저네임(username), 이름(name)을 입력해야 한다.
    - 사용자가 사이트에 가입하면 사용자 번호(id)와 초기 역할(role: USER), 기본 플랜이 부여된다.
    - 이메일과 유저네임은 시스템 내에서 중복될 수 없다.
- 관리자는 사용자 관리 기능을 통해 특정 계정을 잠금(LOCKED) 처리하거나 삭제(WITHDRAWN)할 수 있다.

## 사용자 플랜(plans)
- 사용자 플랜은 아래 정보를 유지해야 한다.
    - 플랜 번호(id)
    - 플랜 이름(name)
    - 월간 제공 토큰량(limit_tokens)
    - 접근 가능 AI 모델 리스트(available_models)
    - 가격(price)
    - 생성 일시(created_at)
- 플랜은 플랜 번호(id)로 식별한다.
- 사용자는 본인의 요구에 따라 플랜을 변경할 수 있으며, 변경 시 가상 결제 시스템을 통해 승인된다.
- 사용자의 플랜 등급에 따라 질문 가능한 AI 모델의 종류와 사용 가능한 토큰 한도가 결정된다.

## AI 채팅 세션(chat_sessions)
- 채팅 세션은 아래 정보를 유지해야 한다.
    - 세션 번호(id)
    - 사용자 번호(user_id)
    - 제목(title)
    - 생성 일시(created_at)
    - 수정 일시(updated_at)
- 채팅 세션은 세션 번호(id)로 식별한다.
- 하나의 세션 안에서 사용자와 AI는 여러 번의 대화를 주고받을 수 있다.
- 사용자는 본인의 채팅 내역(세션)을 자유롭게 삭제할 수 있다.

## 채팅 메시지(chat_messages)
- 채팅 메시지는 아래 정보를 유지해야 한다.
    - 메시지 번호(id)
    - 세션 번호(session_id)
    - 발화 주체(role)
    - 메시지 내용(content)
    - 사용된 AI 모델명(model_name)
    - 소모 토큰량(used_tokens)
    - 생성 일시(created_at)
- 채팅 메시지는 메시지 번호(id)로 식별한다.
- 모든 질문과 답변은 소모된 토큰량과 함께 데이터베이스에 저장된다.
- 사용자는 본인의 잔여 토큰 한도 내에서만 AI에게 질문을 보낼 수 있다.

## 가상 결제 내역(payments)
- 결제 내역은 아래 정보를 유지해야 한다.
    - 결제 번호(id)
    - 사용자 번호(user_id)
    - 플랜 번호(plan_id)
    - 결제 금액(amount)
    - 결제 상태(status)
    - 결제 일시(created_at)
- 결제 내역은 결제 번호(id)로 식별한다.
- 사용자가 플랜을 변경할 때마다 해당 내역이 기록되며, 관리자는 이를 통해 매출 통계를 확인할 수 있다.
